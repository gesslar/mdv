Fedora Machine Sync Guide: Liam & James

This guide will help you configure your two Fedora 43 machines to work seamlessly together.

Machine Details:

  • Liam (Main): 192.168.2.74 (current machine)
  • James: 192.168.2.69

Table of Contents

  1. SSH Setup & Passwordless Login
  2. Remote Desktop (RDP) Setup
  3. Network File Sharing (NFS)
  4. Alternative: SSHFS for File Mounting
  5. Syncing Files with Rsync
  6. Optional: Syncthing for Automatic Sync

1. SSH Setup & Passwordless Login

Since you can already SSH to James, let's make it passwordless for convenience.

On Liam (current machine)

# Generate SSH key if you don't have one
ssh-keygen -t ed25519 -C "liam-to-james"

# Press Enter for all prompts to accept defaults

# Copy your public key to James
ssh-copy-id username@192.168.2.69

# Test passwordless login
ssh username@192.168.2.69

On James (repeat the process in reverse)

# Generate SSH key
ssh-keygen -t ed25519 -C "james-to-liam"

# Copy public key to Liam
ssh-copy-id username@192.168.2.74

Add hostname aliases for easier access

Edit /etc/hosts on both machines:

sudo nano /etc/hosts

Add these lines:

192.168.2.74    liam
192.168.2.69    james

Now you can use ssh james instead of ssh 192.168.2.69

2. Remote Desktop (RDP) Setup

We'll set up RDP using xrdp, which works well with Fedora.

On James (the remote machine)

# Install xrdp
sudo dnf install xrdp xorgxrdp

# Start and enable xrdp service
sudo systemctl start xrdp
sudo systemctl enable xrdp

# Open firewall
sudo firewall-cmd --permanent --add-port=3389/tcp
sudo firewall-cmd --reload

On Liam (to connect to James)

# Install RDP client
sudo dnf install remmina remmina-plugins-rdp

# Launch Remmina
remmina

In Remmina:

  1. Click the "+" button to add a new connection
  2. Protocol: RDP
  3. Server: james (or 192.168.2.69)
  4. Username: your username on James
  5. Password: your password on James
  6. Color depth: True color (24 bpp)
  7. Save and connect!

Repeat on James to RDP back to Liam

Install xrdp on Liam using the same commands above, then configure Remmina on James.

3. Network File Sharing (NFS)

NFS is fast and native to Linux. We'll set up bidirectional sharing.

On James (NFS Server for sharing James folders to Liam)

# Install NFS server
sudo dnf install nfs-utils

# Create a directory to share (or use existing - can share your whole home dir)
mkdir -p /home/username/shared
# Or to share everything: we'll export /home/username directly

Edit NFS exports:

sudo nano /etc/exports

Add this line (replace username with your actual username):

/home/username 192.168.2.74(rw,sync,no_subtree_check,no_root_squash)

Or to share with entire local network:

/home/username 192.168.2.0/24(rw,sync,no_subtree_check,no_root_squash)

# Start and enable NFS
sudo systemctl enable --now nfs-server

# Export the shares
sudo exportfs -arv

# Open firewall
sudo firewall-cmd --permanent --add-service=nfs
sudo firewall-cmd --permanent --add-service=rpc-bind
sudo firewall-cmd --permanent --add-service=mountd
sudo firewall-cmd --reload

On Liam (NFS Client to mount James)

# Install NFS client utilities
sudo dnf install nfs-utils

# Create mount point
sudo mkdir -p /mnt/james

# Mount James's home directory
sudo mount -t nfs james:/home/username /mnt/james

# Make it permanent - edit /etc/fstab
sudo nano /etc/fstab

Add this line:

james:/home/username    /mnt/james    nfs    defaults    0 0

# Test the fstab entry
sudo mount -a

# Verify
ls /mnt/james

Repeat for Liam sharing to James

Follow the same steps but reverse - edit /etc/exports on Liam and create /mnt/liam on James.

4. Alternative: SSHFS for File Mounting

SSHFS is much easier to set up than NFS and requires no server configuration.

On Liam (to mount James)

# Install SSHFS
sudo dnf install fuse-sshfs

# Create mount point
mkdir -p ~/james

# Mount James's home directory
sshfs username@james:/home/username ~/james

# To unmount
fusermount -u ~/james

Make SSHFS mount automatic on login

Add to ~/.bashrc:

nano ~/.bashrc

Add at the end:

# Auto-mount James
if ! mountpoint -q ~/james; then
    sshfs username@james:/home/username ~/james 2>/dev/null
fi

On James (to mount Liam)

sudo dnf install fuse-sshfs
mkdir -p ~/liam
sshfs username@liam:/home/username ~/liam

Add to James's ~/.bashrc:

# Auto-mount Liam
if ! mountpoint -q ~/liam; then
    sshfs username@liam:/home/username ~/liam 2>/dev/null
fi

SSHFS is recommended - it's simpler and just works!

5. Syncing Files with Rsync

For one-time or scheduled syncs between machines.

Basic rsync commands

# Sync a folder from Liam to James
rsync -avz --progress ~/Documents/ james:~/Documents/

# Sync from James to Liam
rsync -avz --progress james:~/Documents/ ~/Documents/

# Mirror (delete files on destination that don't exist on source)
rsync -avz --progress --delete ~/Documents/ james:~/Documents/

# Dry run (see what would be synced)
rsync -avzn ~/Documents/ james:~/Documents/

Flags explained:

  • -a: archive mode (preserves everything)
  • -v: verbose
  • -z: compress during transfer
  • --progress: show progress
  • --delete: mirror exactly (removes extra files)
  • -n: dry run (test first!)

Create a sync script

nano ~/sync-to-james.sh

Add:

#!/bin/bash
echo "Syncing to James..."
rsync -avz --progress --delete \
    ~/Documents/ james:~/Documents/ \
    ~/Pictures/ james:~/Pictures/ \
    ~/Music/ james:~/Music/

echo "Sync complete!"

# Make it executable
chmod +x ~/sync-to-james.sh

# Run it
./sync-to-james.sh

Bidirectional sync script

nano ~/sync-both-ways.sh

Add:

#!/bin/bash
echo "Syncing Liam -> James..."
rsync -avz --progress ~/Documents/ james:~/Documents/

echo "Syncing James -> Liam..."
rsync -avz --progress james:~/Pictures/ ~/Pictures/

echo "Done!"

6. Optional: Syncthing for Automatic Sync

Syncthing provides continuous, automatic, bidirectional file synchronization. Set it and forget it.

On both Liam and James

# Install Syncthing
sudo dnf install syncthing

# Enable and start for your user
systemctl --user enable syncthing
systemctl --user start syncthing

# Open firewall
sudo firewall-cmd --permanent --add-port=22000/tcp
sudo firewall-cmd --permanent --add-port=21027/udp
sudo firewall-cmd --reload

Configure Syncthing

  1. On Liam, open browser: http://localhost:8384
  2. On James, open browser: http://localhost:8384
  3. On each machine, go to Actions → Show ID
  4. On Liam, click "Add Remote Device" and paste James's ID
  5. On James, you'll get a popup to accept Liam's connection - click Add
  6. On Liam, click "Add Folder", select what to sync (e.g., ~/Documents)
  7. In the folder settings, go to Sharing tab and check the box for James
  8. Repeat on James for any folders you want to sync back

Syncthing will now keep selected folders in sync automatically in real-time!

Quick Reference Commands

SSH Connections

ssh james          # Connect to James
ssh liam           # Connect to Liam (from James)

RDP Connections

remmina            # Launch RDP client

NFS Mounts

ls /mnt/james      # View James's files (on Liam)
ls /mnt/liam       # View Liam's files (on James)

SSHFS

sshfs james:/home/username ~/james     # Mount
fusermount -u ~/james                  # Unmount
ls ~/james                             # Browse James's files

File Sync

rsync -avz ~/folder/ james:~/folder/        # Sync to James
./sync-to-james.sh                          # Run sync script

Troubleshooting

Can't connect via RDP

# On James, check xrdp is running
sudo systemctl status xrdp

# Restart it
sudo systemctl restart xrdp

# Check if port 3389 is listening
sudo ss -tlnp | grep 3389

NFS mount fails

# On James, check NFS is running
sudo systemctl status nfs-server

# Check what's being exported
sudo exportfs -v

# On Liam, see what James is sharing
showmount -e james

# Remount
sudo umount /mnt/james
sudo mount -a

SSHFS mount fails

# Check SSH works first
ssh james

# Unmount and remount
fusermount -u ~/james
sshfs username@james:/home/username ~/james

# Enable debug mode to see what's wrong
sshfs -d username@james:/home/username ~/james

Firewall blocking something

# Check what's allowed
sudo firewall-cmd --list-all

# Temporarily disable to test
sudo systemctl stop firewalld

# If that fixes it, add the right rules
sudo firewall-cmd --permanent --add-service=SERVICE_NAME
sudo firewall-cmd --reload
sudo systemctl start firewalld

Recommended Setup

Here's the simplest, most practical setup:

  1. SSH passwordless login - Do this first (Section 1)
  2. SSHFS for file access - Much easier than NFS (Section 4)
    • Mount James at ~/james on Liam
    • Mount Liam at ~/liam on James
    • Now you can access both machines' files from either one!
  3. RDP for graphical access - When you need James's desktop (Section 2)
  4. Syncthing - If you want specific folders to auto-sync (Section 6)

Skip NFS and manual rsync unless you have specific needs for them.

What You'll Be Able To Do

After setup:

✅ SSH to either machine without password: ssh james ✅ Browse James's files on Liam: cd ~/james ✅ Browse Liam's files on James: cd ~/liam ✅ Open James's desktop from Liam (or vice versa) with Remmina ✅ Auto-sync folders between machines with Syncthing ✅ Copy files: cp ~/file ~/james/file (copies from Liam to James!)

Basically, they'll work like one big machine!
